# 工作需求笔记格式规范

## 一、Frontmatter 规范

每个笔记**必须**包含以下 Frontmatter：

```yaml
---
title: 笔记标题
date: YYYY-MM-DD
status: 进行中 | 已完成 | 待处理
tags:
  - 类型标签
  - 主题标签
---
```

### 标签命名规范

| 类型 | 标签示例 |
| ---- | -------- |
| 需求 | `需求` |
| 上线 | `上线` |
| 重构 | `重构` |
| 分析 | `分析` |
| 通知 | `通知` |
| 主题 | `iOS`、`Android`、`Adjust`、`AppsFlyer`、`微信` 等 |

---

## 二、标题层级规范

| 层级 | 用途 | 示例 |
| ---- | ---- | ---- |
| H1 `#` | 笔记主标题 | `# AF归因问题分析` |
| H2 `##` | 主要章节 | `## 问题描述`、`## 解决方案` |
| H3 `###` | 子章节 | `### 1. ATT授权率偏低` |

> [!WARNING] 注意
> 避免使用四级以上标题，改用**加粗**或**表格**呈现信息。

---

## 三、Callout 规范

| 类型 | 用途 | 示例 |
| ---- | ---- | ---- |
| `> [!TIP]` | 提示、建议、参考文档 | `> [!TIP] 参考文档` |
| `> [!WARNING]` | 警告、影响范围 | `> [!WARNING] 影响范围` |
| `> [!IMPORTANT]` | 重要信息、核心问题 | `> [!IMPORTANT] 问题` |
| `> [!INFO]` | 一般信息 | `> [!INFO] 说明` |
| `> [!NOTE]` | 备注 | `> [!NOTE] 备注` |

---

## 四、链接规范

### 附件引用

```markdown
![[attachments/文件名.xlsx]]
![[attachments/文件名.docx]]
```

### 外部链接

```markdown
[链接文字](URL)
```

或使用表格：

```markdown
| 文档 | 链接 |
| ---- | ---- |
| 标题 | [URL](URL) |
```

---

## 五、信息呈现规范

### 基本信息 → 使用表格

```markdown
| 项目 | 内容 |
| ---- | ---- |
| 需求方 | 运营部-张三 |
| 状态 | ✅ 已完成 |
```

### 步骤/流程 → 使用有序列表或表格

```markdown
| 步骤 | 操作 | 状态 |
| ---- | ---- | ---- |
| 1 | 迁移数据 | ✅ |
| 2 | 清理配置 | ⌛ |
```

### 待办事项 → 使用 Checkbox

```markdown
- [x] 已完成项
- [ ] 待处理项
```

## 六、笔记模板

### 需求类笔记模板

```markdown
---
title: 需求标题
date: YYYY-MM-DD
status: 进行中
tags:
  - 需求
  - 主题标签
---

# 需求标题

## 基本信息

| 项目 | 内容 |
| ---- | ---- |
| 需求方 | 部门-姓名 |
| 状态 | 进行中 |

---

## 需求描述

> [!TIP] 背景
> 需求背景说明

---

## 相关文档

- 文档：![[attachments/xxx]]

---

## 待办

- [ ] 待办项1
- [ ] 待办项2
```

### 上线类笔记模板

```markdown
---
title: 项目上线
date: YYYY-MM-DD
status: 已完成
tags:
  - 上线
  - 项目名
---

# 项目上线

## 基本信息

| 项目 | 内容 |
| ---- | ---- |
| 项目名称 | xxx |
| 负责团队 | x组 |

---

## 关键时间节点

| 阶段 | 时间 | 说明 |
| ---- | ---- | ---- |
| 上线 | YYYY-MM-DD HH:mm | 说明 |

---

## 上线渠道

| 渠道 | 渠道ID | 状态 |
| ---- | ------ | ---- |
| xxx | 123 | ✅ |

---

## 待办

- [x] 已完成
- [ ] 待处理
```

```markdown
---

### 1. 原因一

说明...

### 2. 原因二

说明...

---

## 解决方案

| 方案 | 说明 | 推荐 |
| ---- | ---- | ---- |
| 方案1 | xxx | |
| 方案2 | xxx | ⭐ |

---

## 待办

- [ ] 待办项
```

## 七、文件命名规范

格式：`YYYY.MM.DD_类型_主题.md`

| 类型 | 示例 |
| ---- | ---- |
| 需求 | `2026.03.19_需求_越南事件上报.md` |
| 上线 | `2026.03.17_上线_天使俄罗斯.md` |
| 分析 | `2026.04.08_分析_AF归因问题.md` |
| 重构 | `2025.12.08_重构_Redis迁移.md` |
| 通知 | `2026.03.11_通知_Adjust更新.md` |
