# AF归因只有系列数据问题分析

## 问题描述

> [!WARNING] 影响范围
> - **应用ID**：id6752315686、id6752318956、id6752319003
> - **平台**：仅 iOS，Android 正常

**现象**：iOS Google Ads 渠道（Media_Source 为 `googleadwords_int`）的归因数据只有 campaign 层级，缺少 adgroup、ad、channel 字段数据。

---

## 根因分析

### 1. ATT 授权率偏低

| 指标 | 数值 |
| ---- | ---- |
| ATT 授权同意比例 | **12.91%**（4月7日） |
| 携带确定性匹配 ID 信号比例 | **3%** |
| PMOD 概率归因模式占比 | **100%** |

### 2. PMOD 模式限制

PMOD（Probabilistic Matching）模式下：
- Google 仅返还 **campaign 层级** 信息
- 不返还 adgroup、ad、channel 数据

> [!TIP] 参考文档
> [Google ICM traits and limitations](https://support.appsflyer.com/hc/en-us/articles/115002504686-Google-Ads-AdWords-Integration-setup-for-advertisers#google-icm-traits-and-limitations)

### 3. GDPR 配置问题

> [!IMPORTANT] 问题
> 非 EEA 地区用户被错误上报为 `isUserSubjectToGDPR: true`，导致 Google 不进行安装认领。

**解决方案**：非 EEA 地区用户应上报 `isUserSubjectToGDPR: false` 或 `nil`

---

## 对比分析

| 归因方式 | Adjust | AppsFlyer |
| -------- | ------ | --------- |
| 传统归因 | ~70%量级（日本地区） | 0% |
| 概率归因 (PMOD/ICM) | ~30%，显示为 unknown | 100% |
| adgroup 数据 | 有 | 无 |

---

## 解决方案

### 方案对比

| 配置项 | 方案1（原方案） | 方案2（推荐方案） |
| ------ | -------------- | ----------------- |
| AF 初始化时机 | ATT 授权后 | didFinishLaunching 中（提前） |
| waitATT | 60s | 30-60s（平衡数据 gap 和 IDFA 收集率） |
| isUserSubjectToGDPR | @YES（EEA 用户） | nil（非 EEA 用户） |
| hasConsentForDataUsage | @NO | nil |

> [!TIP] AF 推荐方案2的原因
> - AF 初始化提前可更早开始归因流程
> - waitATT 时间过长（60s）容易引起数据 gap
> - 非 EEA 用户 GDPR 参数应为 nil 而非 @NO

---

### 最终采用配置

#### 1. 初始化流程

```objc
// didFinishLaunching 中
// 1. 先设置debug（必须在start前）
[AppsFlyerLib shared].isDebug = true;

// 2. 设置waitATT（延迟启动等待ATT授权）
[[AppsFlyerLib shared] waitForATTUserAuthorizationWithTimeoutInterval:10];

// 3. 启动SDK
[[AppsFlyerLib shared] start];
```

#### 2. ATT 弹窗

```objc
// JP_SDKManager.requestTrackingAuthorization
- (void)requestTrackingAuthorization:(void(^)(void))block {
    if (@available(iOS 14, *)) {
        [ATTrackingManager requestTrackingAuthorizationWithCompletionHandler:^(ATTrackingManagerAuthorizationStatus status) {
            // 授权回调处理
        }];
    }
}
```

#### 3. GDPR/Consent 设置

```objc
// 非EEA用户推荐配置
AppsFlyerConsent *consent = [[AppsFlyerConsent alloc]
    initWithIsUserSubjectToGDPR:nil      // 关键：用nil而非@NO
    hasConsentForDataUsage:nil
    hasConsentForAdsPersonalization:nil
    hasConsentForAdStorage:nil];
```

#### 4. SDK 版本

已升级至 **6.17.9**

### 2026/4/13

**新工单 #952154**：请求 AF 提供代码级别的 SDK 对接支持

**AF 回复**：解释 `waitForATTUserAuthorizationWithTimeoutInterval` 的作用
- 调用此 API 有延迟 start 调用的功能
- 若未调用，start 可能在 ATT 弹窗弹出时已执行，导致归因流程启动时未收集到 IDFA
- AF 向 Google 发送 ping 时就没有携带 IDFA

- AF 配置：`TSAppsFlyerConfigurator`
