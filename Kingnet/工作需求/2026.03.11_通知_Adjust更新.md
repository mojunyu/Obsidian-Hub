# Adjust Redownload 归因逻辑更新

> [!IMPORTANT] 重要通知
> 自 3 月 16 日起，Adjust 将开始在归因、回传和报告中，将符合条件的应用重装（Redownload）记录为新的安装。

---

## 变更原因

目前，Adjust 在一台设备的生命周期内仅记录一次首次安装，并将其永久归因于最初带来安装的渠道。

这种方法未能像预期那样，通过为广告渠道的模型提供有价值的信号来最大化付费媒体的投放规模。因此，Adjust 决定更改逻辑：**今后将把符合条件的重装视为带有新归因来源的全新安装。**

---

## 工作原理

### Redownload 触发条件

必须同时满足以下三个要求：

1. Adjust SDK 应在应用从设备上被删除后追踪到一次会话
2. 自上次安装以来，已过去足够的时间（由「Redownload 排除窗口期」控制）
3. 用户在应用中长时间处于非活跃状态（由「Redownload 非活跃期」控制）

### Redownload Install 符合条件时

1. 在归因、原始数据和报告中被视为一次新的安装
2. 归因流程遵循与常规安装相同的逻辑，后备归因渠道为「自然量」
3. 生成新的安装活动，反映给您和渠道的回传以及汇总报告中
4. 设备历史同期群表现重置，从 Redownload 开始重新计算
5. 设备标识保持不变：`adid` 不会因为 Redownload 而重置

## 操作建议

### 1. 调整设置

Redownload 归因逻辑将于 3 月 16 日生效，当前设置入口已开放。

**设置路径**：AppView → 归因设置 → Redownload 设置

| 配置项 | 可调范围 | 默认值 | 说明 |
| ------ | -------- | ------ | ---- |
| Redownload 排除窗口期 | 1-24 个月 | 3 个月 | 窗口期越长，Redownload Install 数据越少 |
| Redownload 非活跃期 | 1-30 天 | 14 天 | 非活跃期越长，Redownload Install 数据越少 |

> [!TIP] 提示
> 如希望保持和目前近似的效果，可将排除窗口期设置为 12 个月及以上。

### 2. 新增占位符

可将以下占位符添加到回传和 CSV 上传设置中：

| 占位符 | 说明 |
| ------ | ---- |
| `{is_redownload}` | 该安装是否为 Redownload |
| `{redownload_window}` | Redownload 排除窗口期 |
| `{redownload_inactivity_period}` | Redownload 非活跃期 |

### 3. 新指标

新的 Redownload 指标可在 Datascape 报告中查看所有与 Redownload 相关的行为。
