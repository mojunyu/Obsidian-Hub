### 3️⃣新建config文件

在 ~/.ssh 目录下，创建config文件，并写上配置项、

**# 9130**：

- `Host`：这是一个自定义的标识符，用于指定你要连接的远程主机。在这个例子中，`Host` 是 `192.168.40.11`，这意味着这个配置将应用于连接到 IP 地址为 `192.168.40.11` 的主机。
- `HostName`：这是实际的主机名或 IP 地址，指定了要连接到的远程主机的地址。在这里，`HostName` 是 `192.168.40.11`，这是一个局域网中的主机。
- `User`：这是登录到远程主机时要使用的用户名。在这里，`User` 是 `mojunyu`。
- `IdentityFile`：这是 SSH 密钥文件的路径，用于身份验证。在这个配置中，SSH 将使用 `~/.ssh/9130/id_rsa_2048` 文件作为私钥。

![image-20240508154654893](./image-20240508154654893.png)

**# github**：

- `Host`：同样是一个自定义的标识符，用于指定连接的远程主机。在这个配置中，`Host` 是 `github.com`。
- `HostName`：这是实际的主机名或 IP 地址，指定了要连接到的远程主机的地址。对于 GitHub，`HostName` 是 `github.com`。
- `User`：这是登录到远程主机时要使用的用户名。在连接 GitHub 时，通常使用的用户名是你的 GitHub 用户名。
- `IdentityFile`：这是 SSH 密钥文件的路径，用于身份验证。在这个配置中，SSH 将使用 `~/.ssh/github/id_rsa` 文件作为私钥。



### 4️⃣删除known_hosts



`如果还是拉取失败，极有可能是之前生成了错误的密钥算法，删掉对应的host条目，重新拉取，让known_hosts重新生成`



`known_hosts` 文件是 SSH 客户端用来存储已知主机的公钥信息的文件。当你第一次连接一个新的远程主机时，SSH 客户端会询问你是否信任该主机的公钥，如果你选择信任，那么该主机的公钥将会被保存到 `known_hosts` 文件中。

```sh
[主机名]:端口号 公钥算法 公钥内容
```

`known_hosts` 文件的内容是在你第一次连接远程主机时生成的。当你选择信任远程主机的公钥时，SSH 客户端会将远程主机的公钥信息添加到 `known_hosts` 文件中。以后再次连接相同主机时，SSH 客户端会检查 `known_hosts` 文件，以确保远程主机的公钥与之前保存的一致。如果公钥不一致，SSH 客户端会发出警告，提醒你可能连接到了不安全的主机。
